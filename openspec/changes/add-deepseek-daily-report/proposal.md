# Add DeepSeek Daily Report Generation

## Why
The current daily report is generated by deterministic local rules. Users need an optional true AI report path that can call DeepSeek with a user-provided API key, without downloading or hosting a local model.

## What Changes
- Add a DeepSeek-backed daily report endpoint.
- Accept the DeepSeek API key in the request body for that call only.
- Reuse the existing local daily report aggregation as structured context.
- Return the same daily report shape plus AI metadata.
- Keep the existing local `GET /reports/daily` endpoint as the offline fallback.

## Non-Goals
- Do not store DeepSeek API keys in the database or server config.
- Do not download or run local LLM weights.
- Do not replace the deterministic local daily report path.

## Impact
- Server report API gains one authenticated POST endpoint.
- Server needs outbound HTTPS access to DeepSeek when the AI endpoint is used.
- Web report page gains a user-entered API key field and manual AI generation action.
